# Правила разработки проекта

## Обязательные правила

### 1. Перезапуск проекта после изменений

**⚠️ ВАЖНО:** После любых изменений в коде проекта необходимо **обязательно перезапустить приложение**.

Это необходимо для того, чтобы:
- Применить изменения в Java классах
- Обновить шаблоны HTML
- Применить изменения в конфигурационных файлах
- Убедиться, что все изменения работают корректно

**Процедура перезапуска:**

1. Остановить текущий процесс на порту 8080:
   ```powershell
   $conn = Get-NetTCPConnection -LocalPort 8080 -State Listen -ErrorAction SilentlyContinue
   if ($conn) { 
       Stop-Process -Id $conn.OwningProcess -Force 
   }
   ```

2. Запустить проект заново:
   ```powershell
   $env:PATH = "C:\Program Files\Java\apache-maven-3.9.11\bin;$env:PATH"
   mvn spring-boot:run
   ```

Или использовать скрипт `run-service.ps1`:
```powershell
.\run-service.ps1
```

**Когда нужно перезапускать:**
- После изменения любого Java класса
- После изменения HTML шаблонов
- После изменения конфигурационных файлов (application.yml и т.д.)
- После добавления новых зависимостей в pom.xml (требуется пересборка)

**Когда НЕ нужно перезапускать:**
- Изменения в документации (.md файлы)
- Изменения в файлах, не используемых приложением

---

## Дополнительные правила

### 2. Документирование изменений

Все значимые изменения должны быть задокументированы:
- В соответствующих .md файлах
- В комментариях к коду
- В commit сообщениях (если используется git)

### 3. Проверка работоспособности

После внесения изменений необходимо:
- Проверить, что приложение запускается без ошибок
- Проверить, что измененная функциональность работает корректно
- Убедиться, что не сломалась существующая функциональность

### 4. Обработка ошибок

- Все ошибки должны обрабатываться корректно
- Пользователю должны показываться понятные сообщения об ошибках
- Технические детали ошибок должны логироваться, но не показываться пользователю
